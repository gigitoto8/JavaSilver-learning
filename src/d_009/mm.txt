違いを一言で
概念	　　　　役割
多態性	　　　　考え方・性質
interface	　多態性を実現する道具

多態性とは？
Animal a = new Dog();
a.speak();
同じ呼び方だが、中身が変わる

interface の役割
Animal a = new Dog();
Animal b = new Cat();
共通の型を与える
実装を縛らない
👉 多態性を成立させる「型の器」

使い分け（実務＆Silver）
interface を使うとき
「できること」を表したい
実装が複数ある
将来増えそう

継承（class extends）を使うとき
「共通の状態（フィールド）」がある
is-a 関係が明確
実装を共有したい

超重要な一文（覚えてOK）
interface は「設計」、継承は「実装共有」




最終整理（Day9の核心）

呼べるメソッド → 型
実際に動く処理 → 実体
多態性 → 実体差し替え
interface → 差し替え可能にする仕組み

interface
・「できること」
・多態性の器
・設計のため

継承
・「何者か」
・実装共有
・再利用のため

多態性
・同じ呼び方
・中身が違う
